# Topic Prompt: Warehouse Inventory System

Design a medium-size warehouse inventory management system for a regional retailer with one main warehouse and several smaller forward-stock locations.

The system should support receiving, putaway, inventory lookup, reservations, picking, packing, shipping, cycle counts, adjustments, returns, and inventory reporting. Users include warehouse associates, supervisors, inventory control staff, customer support, and integration operators.

Important conditions:

- Inventory is scanned with barcode devices, but manual correction must be possible with audit trails.
- Stock can be reserved for orders, transferred between locations, damaged, quarantined, returned, or adjusted after cycle counts.
- The system integrates with order management, carrier systems, procurement, and finance reporting.
- The operator needs a plan that can later produce implementation tasks, test work, integration work, and rollout work.

Use matched abstraction and stop at Plan.
