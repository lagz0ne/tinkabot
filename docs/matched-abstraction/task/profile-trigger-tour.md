# Profile Trigger Tour Task

## Brief

Tinkalet edge slice 1, per `../plan/tinkalet-edge.md`: create the first
product-shaped user path over the existing Tinkabot daemon. A user can import
or select a local profile, then trigger an existing Tinkabot intent such as the
clock bundle without learning the NATS CLI, raw subject names, or credential
file layout first.

This Task is the pre-code artifact pack for that slice. It is intentionally
bounded to the tour bridge:

- profile import/use/list for a local Tinkabot store;
- one product trigger command over the active profile;
- Slice 1 README/example clock-tour blocks backed by package smoke;
- raw NATS command retained as proof and diagnostic surface.

It does not own durable item records, waits, watches, local reactions, schedule
records, remote pairing, or a long-running Tinkalet daemon.

This document is the Slice 1 umbrella Task. Implementation must descend into
bounded sub-Tasks before code:

1. `tinkalet-cli-profile`: `cmd/tinkalet`, the profile config model, and
   import/list/use/default behavior over a fixture or already-materialized
   descriptor; config permissions, no user-home mutation, output/JSON oracles.
   It must not require Tinkabot to emit `<store>/local-profile.json`; that is
   sub-Task 2.
2. `tinkalet-local-profile-source`: Tinkabot emits the local profile descriptor
   and Tinkalet imports it from `--store`.
3. `tinkalet-trigger-live`: active-profile trigger over real embedded NATS,
   visible clock state change, request-id duplicate proof.
4. `tinkalet-authority-denials`: revoked/stale/denied-neighbor/no-stronger-creds
   fallback and output privacy.
5. `tinkalet-package-tour`: archive contains both binaries and runs the
   source/archive tour with scrubbed `PATH`.

Each sub-Task must have its own RED artifact and concrete verification evidence.

## Artifact Pack

### User Flow

```text
package user
    |
    | starts existing daemon with clock bundle
    v
tinkabot --store <dir> --bundle examples/clock
    |
    | writes local role creds and prints/listens on loopback
    v
tinkalet profile import local --store <dir> --name local
    |
    | stores profile metadata under Tinkalet config
    v
tinkalet profile use local
    |
    | sets default profile
    v
tinkalet trigger bundle.clock.tick
    |
    | resolves endpoint + caller creds + derived trigger subject
    v
Tinkabot bundle trigger
    |
    v
accepted / duplicate / denied response in product language
```

### Local Profile Source Contract

`tinkalet profile import local --store <dir>` must not depend on the user
copying a printed NATS URL or a raw credential path. The implementation must
choose and test one machine-readable source of local profile metadata:

```text
Tinkabot local profile descriptor
  server endpoint
  shell endpoint when available
  caller or edge credential reference
  role name used for trigger requests
  trust class: local-owner
  source: local-store:<dir>
```

For Slice 1 RED tests, the machine-readable source is fixed as:

```text
<tinkabot-store>/local-profile.json
```

Minimum JSON shape:

```json
{
  "kind": "tinkabot.localProfile.v1",
  "server": "nats://127.0.0.1:<port>",
  "shell": "http://127.0.0.1:<port>",
  "credential": "caller.creds",
  "role": "caller",
  "trust": "local-owner",
  "source": "local-store:<absolute-store-dir>"
}
```

Tests must fail import when this descriptor is absent, malformed, points at a
default port not actually served by the daemon, or points outside the store.
Human posture output is never a valid fallback source.

`profile import local` copies `<store>/<descriptor.credential>` to
`<TINKALET_DATA_DIR>/profiles/<name>/caller.creds` with mode `0600`.
`profiles.json` stores `credentialRef` as `profiles/<name>/caller.creds`.
Import fails with `import-source-invalid` if the descriptor credential is
absolute, escapes the store, or the source credential file is missing. If the
managed copy is deleted after import, trigger fails with `stale-credentials`.

### Profile Storage Contract

```text
Tinkalet config
  profiles.json        mode 0600 when created
  default-profile      mode 0600 when created

profile record
  name
  server endpoint
  shell endpoint when available
  managed credential copy path relative to TINKALET_DATA_DIR
  credential role: caller or edge
  trust class
  source
```

Normal output must never print credential contents, NKEY/JWT material, operator
keys, or stronger role credentials. `--json` may include diagnostic paths and
profile metadata, but it must redact secret contents.

### Command Contract

The first tour command grammar is:

```text
tinkalet profile import local --store <dir> --name local
tinkalet profile list
tinkalet profile list --json
tinkalet profile use local
tinkalet trigger bundle.clock.tick
tinkalet trigger bundle.clock.tick --request-id req-tour-1
tinkalet trigger bundle.clock.tick --profile local --json
```

Profile command output oracle:

```text
profile import success: stdout exactly "profile local imported\n", stderr empty, exit 0
profile import missing source: stdout empty, stderr exactly "profile local denied profile import: import-source-missing\n", exit 1
profile import invalid source: stdout empty, stderr exactly "profile local denied profile import: import-source-invalid\n", exit 1
profile list with no profiles: stdout exactly "no profiles\n", stderr empty, exit 0
profile list after local imported but no default: stdout exactly "- local caller local-owner\n", stderr empty, exit 0
profile list after local selected: stdout exactly "* local caller local-owner\n", stderr empty, exit 0
profile use success: stdout exactly "profile local selected\n", stderr empty, exit 0
profile use missing: stdout empty, stderr exactly "profile missing denied profile use: profile-not-found\n", exit 1
trigger with no default: stdout empty, stderr exactly "profile default denied bundle.clock.tick: profile-not-found\n", exit 1
```

`profile list` prints one line per profile, sorted by profile name. The first
column is `*` for the default profile and `-` for a non-default profile.

`profile list --json` writes one JSON document plus a trailing newline to
stdout, writes nothing to stderr, and exits 0. Tests should decode the JSON
rather than depend on whitespace. The decoded shape is:

```json
{
  "default": "local",
  "profiles": [
    {
      "name": "local",
      "default": true,
      "server": "nats://127.0.0.1:<port>",
      "shell": "http://127.0.0.1:<port>",
      "role": "caller",
      "trust": "local-owner",
      "source": "local-store:<absolute-store-dir>",
      "credentialRef": "profiles/local/caller.creds",
      "credentialRedacted": true
    }
  ]
}
```

With no profiles, `profile list --json` decodes as:

```json
{
  "default": "",
  "profiles": []
}
```

With an imported profile but no default selected, `profile list --json` decodes
as:

```json
{
  "default": "",
  "profiles": [
    {
      "name": "local",
      "default": false,
      "server": "nats://127.0.0.1:<port>",
      "shell": "http://127.0.0.1:<port>",
      "role": "caller",
      "trust": "local-owner",
      "source": "local-store:<absolute-store-dir>",
      "credentialRef": "profiles/local/caller.creds",
      "credentialRedacted": true
    }
  ]
}
```

Profile JSON arrays are sorted by profile name. `credentialRef` is relative to
the Tinkalet data dir; JSON output must never include credential contents,
NKEY/JWT text, operator key paths, or stronger-role credential references.

Trigger output should be product-shaped:

```text
profile local accepted bundle.clock.tick
profile local duplicate bundle.clock.tick
profile local denied bundle.clock.tick: <reason-code>
```

Normal output oracle:

```text
accepted: stdout exactly "profile local accepted bundle.clock.tick\n", stderr empty, exit 0
duplicate: stdout exactly "profile local duplicate bundle.clock.tick\n", stderr empty, exit 0
expected denial: stdout empty, stderr exactly "profile <name> denied <intent>: <reason-code>\n", exit 1
usage or parse error: stdout empty, stderr starts with "usage: tinkalet ", exit 2
```

`<reason-code>` must be one of the Failure Matrix result codes, not a raw NATS
error string.

The command may still expose diagnostic details with `--json`; the normal
success path should not require users to know `tb.bundle.clock.tick`.
`bundle.clock.tick` is the one taught Slice 1 intent; broader trigger discovery
may remain shallow. A product-shaped unknown-trigger error must point the user
back to the documented tour intent with the `unknown-trigger` reason code and
without exposing raw NATS subjects.

JSON output oracle:

```json
{
  "profile": "local",
  "intent": "bundle.clock.tick",
  "status": "accepted",
  "reason": "",
  "requestId": "req-tour-1",
  "diagnostics": {
    "server": "nats://127.0.0.1:<port>",
    "shell": "http://127.0.0.1:<port>"
  }
}
```

`status` is one of `accepted`, `duplicate`, or `denied`. `reason` is empty for
accepted and duplicate responses; for denials it must be one of the Failure
Matrix result codes. `diagnostics` may include paths and raw subjects only
under `--json`; it must never include credential contents, NKEY/JWT material,
operator keys, or NATS permission dumps.

### Failure Matrix

```text
+------------------+----------------------------+--------------------------+
| Family           | Condition                  | Expected product result  |
+------------------+----------------------------+--------------------------+
| missing-profile  | no default or named profile| profile-not-found        |
| bad-source       | store lacks descriptor     | import-source-missing    |
| bad-source-shape | malformed/outside endpoint | import-source-invalid    |
| stale-creds      | creds path missing/expired | stale-credentials        |
| revoked-creds    | server revoked profile cred| revoked-credentials      |
| denied-neighbor  | profile targets wrong auth | denied-neighbor          |
| denied-trigger   | profile lacks publish grant| denied-trigger           |
| strong-fallback  | caller missing, author set | stale-credentials        |
| unreachable      | server listener unavailable| connection-failed        |
| malformed-reply  | non-product reply body     | malformed-response       |
| duplicate        | same request id replay     | duplicate, no rerun      |
| override         | --profile targets another  | override target only     |
| unknown-trigger  | intent not known in slice  | unknown-trigger          |
| output-leak      | raw substrate in normal text| output-leak              |
+------------------+----------------------------+--------------------------+
```

Every failure result must name the selected, requested, or default profile
identifier when one exists; otherwise it must say that no default profile is
configured. It must avoid dumping raw NATS internals unless `--json` is
requested.

Normal output or normal errors must not contain raw `tb.` subjects, `_INBOX`,
credential file contents, NKEY/JWT text, NATS permission dumps, stack traces, or
operator-key paths. `--json` may expose diagnostic subjects and paths only when
secret contents remain redacted.

## Acceptance Contract

This is the Slice 1 umbrella acceptance, not a code-task acceptance contract.
Implementation may start only from a bounded sub-Task document with its own RED
artifact. The first implementation Task is `tinkalet-cli-profile`; it owns only
`cmd/tinkalet` help/parsing, `profile import/list/use`, default lookup,
`--profile` selection behavior, profile JSON/human output, config/data-dir
writes, permissions, and no user-home mutation. It does not own Tinkabot
descriptor emission, real embedded-NATS trigger proof, revocation/neighbor
denials, package archive smoke, or docs gates.

- `go test ./cmd/tinkalet -count=1` from `substrate/go` passes and proves the
  CLI entry surface exists, prints help, parses `profile` and `trigger`
  subcommands, supports `--request-id`, and returns typed product errors for
  malformed commands with stable stdout, stderr, and exit-code behavior.
- A profile unit test proves `import local`, `list`, `use`, default lookup, and
  `--profile` override with temporary config/data dirs, no global CLI, no user
  home mutation, `0600` config files, and no secret material in `profile list`
  or `profile list --json`.
- Profile tests run with `HOME`, `XDG_CONFIG_HOME`, and `XDG_STATE_HOME` set to
  temp directories and explicit `TINKALET_CONFIG_DIR` and `TINKALET_DATA_DIR`
  temp env vars. Tests snapshot those directories before and after command
  execution and fail on writes outside the selected config/data roots.
- A local profile source test starts Tinkabot on a non-default or dynamic
  loopback endpoint, imports only with `--store <dir>`, and proves the profile
  records server endpoint, shell endpoint when available, managed caller/edge
  credential copy path, trust class, and source without asking the user for
  `CLIENT_URL` or a raw creds path.
- A real embedded NATS operator-tour test starts Tinkabot with
  `examples/clock`, imports the local profile, selects it, runs
  `tinkalet trigger bundle.clock.tick`, and observes the clock projection or
  served page state change. A reply of `accepted` alone is not sufficient.
- The state-change oracle is the shell endpoint from the imported profile:
  `GET <shell>/projections/bundle.clock.state` returns a material projection
  JSON document before and after the trigger, and the post-trigger document has
  a larger `sequence` or `value.unix` than the pre-trigger document. The test
  polls long enough to avoid same-second clock collisions. The package-tour
  transcript stores the exact before and after payloads as evidence.
- An idempotency test runs `tinkalet trigger bundle.clock.tick --request-id
  <id>` twice and proves the first result is `accepted`, the second is
  `duplicate`, and the clock state does not advance a second time for the same
  request id.
- The idempotency no-rerun oracle is byte-identical
  `/projections/bundle.clock.state` after the duplicate reply.
- Profile isolation tests use two distinguishable profiles or daemon surfaces:
  default profile A plus `--profile B` advances only B; profile A cannot trigger
  B's neighboring surface; the default remains A after the override.
- Denial tests prove missing profile, bad local profile source, unreachable
  daemon, stale credentials, revoked credentials, denied trigger,
  denied-neighbor, stronger-credential fallback refusal, malformed response,
  duplicate replay, and profile override behavior through product-shaped errors.
- Credential-selection tests prove Tinkalet uses only the selected profile's
  caller or edge credential. If `caller.creds` is missing or invalid while
  `author.creds`, `observer.creds`, or operator key material exists, trigger
  fails as stale/denied instead of falling back to stronger authority.
- Output tests prove normal success and failure output names the profile and
  intent, uses stable product wording, separates stdout/stderr consistently,
  and excludes raw substrate details. Malformed-response tests must reject empty
  bodies, unknown statuses, substring traps such as `accepted-but-not-really`,
  invalid `--json` payloads, and no-responder cases.
- Packaging includes the `tinkalet` binary beside `tinkabot`, `bwrap`, and the
  pinned NATS CLI sidecar without making npm the distribution authority. The
  archive root `release.json` declares archive contents in a non-promotional
  form with at least:

  ```json
  {
    "binary": "tinkabot",
    "commands": ["tinkabot", "tinkalet"],
    "bundledBwrap": "libexec/tinkabot/bwrap",
    "bundledNats": "libexec/tinkabot/nats"
  }
  ```

  Slice 6 owns changing metadata into a public support claim and wiring that
  claim into centralized release evidence.
- A release-archive smoke runs `bun run release:package <tmp>`, unpacks the
  archive, asserts executable `./tinkabot` and `./tinkalet`, runs
  `./tinkalet --help` or `--version`, starts `./tinkabot --bundle
  examples/clock`, runs the profile/trigger tour from the archive root, and
  uses a scrubbed `PATH` so no global `nats` or ambient `tinkalet` can satisfy
  the proof. A separate package smoke disables or renames
  `libexec/tinkabot/nats` while running the Tinkalet profile/trigger happy path,
  proving Tinkalet uses the Go ecosystem directly instead of shelling out to
  the bundled NATS CLI sidecar.
- README and example docs may document the Slice 1 Tinkalet clock tour only
  with an explicit release boundary: the v1 manual and centralized
  `release:evidence` remain the NATS-CLI public authority until Slice 6. Any
  changed command block must execute from the source checkout or unpacked
  archive. Raw NATS commands may remain only as proof/diagnostics. Schedule
  controls remain diagnostic until the schedule slice owns product commands.
- Slice 1 adds standing implementation/scenario coverage for the Tinkalet
  trigger surface, covering allowed, denied-neighbor, malformed, duplicate,
  stale, revoked, and attributed-failure families. It must not add Tinkalet to
  `release/v1.json` or the centralized `release:evidence` required release
  claims. Slice 6 promotes that coverage into public release evidence.
- Before package-tour acceptance, prefer extending `gate:scenarios` with a
  required outside-in surface list that includes `tinkalet-trigger`; use a peer
  gate only if there is a concrete reason `gate:scenarios` cannot own it. The
  gate must fail when that surface is absent, not only when a declared surface
  is incomplete.
- Manual and centralized `release:evidence` migration remain owned by the
  release-docs-and-proof slice before any release claim says Tinkalet is the
  primary public operating surface.

## Scope

Owns:

- the first `tinkalet` CLI entry point;
- profile import/list/use/default behavior for local profiles;
- active-profile and `--profile` trigger resolution;
- product-shaped trigger response rendering;
- docs/package proof for the release-tour path.

Does not own:

- durable item APIs;
- watch cursors;
- local reaction execution;
- schedule records;
- remote pairing;
- long-running Tinkalet daemon behavior;
- changing Tinkabot's bundle authority model.

## Residual Risk

- Local profile import intentionally depends on the local store descriptor
  written by the Tinkabot binary. Remote pairing stays deferred.
- The trigger naming contract may need one refinement after real usage. Later
  slices may refine spelling, but they must preserve product intent names and
  must not make raw NATS subjects the happy path.
- The v1 manual and centralized release evidence still use the NATS CLI as the
  public authority. Promoting Tinkalet as the primary public release surface is
  owned by `release-docs-and-proof`.
